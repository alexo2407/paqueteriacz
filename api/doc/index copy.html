Logistics API Documentation

Complete guide to use our API

1. Authentication (Login)

Use this endpoint to authenticate users and obtain a JWT token for future requests.

Endpoint:

POST /api/auth/login

Request Parameters:

  • email (string): The user's email
  • password (string): The user's password

Example Request Body:

{ "email": "admin@example.com", "password": "123456" }

Example Response:

{ "success": true, "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }

2. Using the JWT Token

After obtaining a valid JWT token, include it in the Authorization header of each request:

Header Example:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

The API will validate the token before processing any requests.

3. CRUD Operations for Orders

3.1 Get All Orders

Retrieve a list of all orders.

Endpoint:

GET /api/pedidos

Headers:

Authorization: Bearer [YOUR_JWT_TOKEN]

Example Response:

[ { "id": 1, "order_number": "12345", "recipient": "John Doe", "phone": "50512345678", "price": 3000, "address": "Main Street #123", "comment": "Urgent delivery", "status": "In warehouse" } ]

3.2 Create an Order

Create a new order in the system.

Endpoint:

POST /api/pedidos

Headers:

Authorization: Bearer [YOUR_JWT_TOKEN]

Request Body:

{ "order_number": "12345", "recipient": "John Doe", "phone": "50512345678", "price": 3000, "address": "Main Street #123", "comment": "Urgent delivery", "latitude": "12.345678", "longitude": "-86.789012" }

Example Response:

{ "success": true, "message": "Order successfully created", "data": { "id": 10, "order_number": "12345" } }
